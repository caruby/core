caTissue Migrator

The caRuby Tissue Migrator imports an input source into a caTissue database. The sole configuration is the migration mapping file and a small caTissue server access properties file. The defaults and other metadata are pre-defined in the caRuby Tissue declarations. These declarations express basic model characteristics, and are expected to be stable and sufficient for most uses. Site-specific migration customization is accomplished in small Ruby shim files. The shim file is well-suited for filters to convert an input field value into a supported caTissue property value. See the Migration FAQ for more information.

The application-independent caRuby Core Migrator provides 80% of the migration functionality. The caRuby Tissue Migrator tailors the Core Migrator to the caTissue data model. An additional tissue-specific feature is tissue site and clinical diagnosis permissible value mapping and validation.
