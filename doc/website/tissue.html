caRuby Tissue

The caRuby Tissue component applies the caRuby facade to the caTissue application. This offers the following features:

  • flexible server interaction
  • data migration and extraction
  • a foundation for a caSmall framework

Since the caRuby Tissue component is built on the caTissue API, all of the caTissue business logic applies. In addition, caRuby augments the API as follows:

  • supplies default attribute values
  • validates mandatory attributes before submission to caTissue
  • validates additional attributes which are permitted by the caTissue API but corrupt the database
  • migrates an input CSV record to a caTissue object based on one simple configuration file
  • validates migration input diagnosis and tissue site permissible values
  • optionally maps diagnosis and tissue site migration input values to caTissue permissible values
  • determines how to save the caTissue object based on the caTissue object model
  • finds referenced objects based on secondary keys and alternative search strategies
  • creates whatever needs to be created in the required order based on the data model associations
  • works around caCORE and caTissue bugs and traps
  • provides helper methods for common tasks, e.g. aliquoting
The migration capability is described in the Migrator page.

There is little class-specific caRuby business logic code similar to that found in caTissue. Rather, there are basic design patterns driven by the model characteristics defined in the Ruby declarations.
